# Virtue of Complexity + QP Optimizer: Full Experimental Results

**Date**: 2026-04-16  
**Runtime**: 77.3 minutes (4638s)  
**Script**: `run_voc_qp.py`

## Methodology

Implementation of Bryan Kelly's "Virtue of Complexity" (Kelly, Malamud & Zhou 2020) and "AI Pricing Theory" (Didisheim, Ke, Kelly, Malamud 2023) for 4H Binance crypto perpetual futures.

### Two Signal Generation Methods

1. **SDF (Stochastic Discount Factor)**: Builds P characteristic-managed long-short factor portfolios, then optimizes Markowitz portfolio with ridge on factor return time series. `c = 2P/T` where T = training window.

2. **FM (Fama-MacBeth)**: Per-bar cross-sectional ridge regression. `c = 2P/(N*W)` where N = ~49 assets, W = pooling window.

Both use **Random Fourier Features (RFF)** to project K=34 characteristics into 2P-dimensional feature space.

### Three Key Innovations Tested

1. **Multi-Horizon Targets (h)**: Instead of predicting 1-bar returns, predict cumulative h-bar returns. Longer horizons produce smoother, more stable signals.

2. **Temporal Ridge Penalty (tau)**: Penalize changes in ridge coefficients between refits: `min ||...|| + tau*||beta - beta_prev||^2`

3. **QP Optimizer**: Convex optimization post-processing with transaction cost penalty, max turnover constraints, and controlled rebalancing frequency.

## Key Conclusions

### 1. Multi-Horizon Target is Critical
FM with h=6 (1-day ahead prediction) achieves test IC of +0.064 vs +0.019 for h=1. The signal is inherently smoother because the prediction target changes slowly.

### 2. QP Optimizer is the Key Enabler
QP reduces turnover by 75-1000x (from ~1.0 to 0.001-0.013) while preserving or improving Sharpe. The best raw signal (FM h=6 SR@5=+2.66) improves to SR@5=+3.77 with QP.

### 3. FM Dominates SDF for Tradeable Alpha
FM generates per-asset predictions that QP can selectively act on. SDF generates portfolio-level weights that clash with QP's own optimization.

### 4. Temporal Ridge Penalty (tau) is Counterproductive
All tau>0 configs perform worse than tau=0 + QP. The QP optimizer handles turnover reduction far more effectively.

### 5. Optimal QP Parameters
- **rebal_every=12** (2 days) beats 24 (4 days) for most configs
- **max_turnover=0.10** slightly edges 0.05 for FM signals
- **tcost_lambda=0.005-0.01** both effective; 0.005 slightly better for h=6
